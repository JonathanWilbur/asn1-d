codec

The codecs in this library are values of an ASN1 encoding scheme. The type, length, and value are set using properties defined on abstract classes from which the encoding-specific values inherit. This module contains those abstract classes, and with these classes, it serves as the root module from which all other codec modules will inherit.

    class AbstractSyntaxNotation1ValueInvalidException: codec.AbstractSyntaxNotation1CodecException;

    Thrown when an encoded value, or a decoded value (attempting to be encoded) takes on a value that the codec cannot encode or decode.

    Examples

    • When a BER codec finds an invalid character in a string

  • AbstractSyntaxNotation1InvalidIndexException

    Declaration

    class AbstractSyntaxNotation1InvalidIndexException: codec.AbstractSyntaxNotation1CodecException;

    An exception thrown when a member of a CHOICE or SEQUENCE is given a context-specific index that is not defined for that CHOICE or SEQUENCE.

    Discussion

    For example, if:

    TheQuestion := [APPLICATION 5] CHOICE { toBe [0] NULL, notToBe [1] NULL }

    This exception should be thrown if TheQuestion were to be decoded from the BER-encoded byte sequence: 0x65 0x02 0x83 0x00, because the third byte specifies a third choice in TheQuestion, but there is no choice #3 in TheQuestion--there is only choice #0 and #1.

    • objectDescriptor

      Declaration

      abstract const @property string objectDescriptor();

      Decodes an ObjectDescriptor, which is a string consisting of only graphical characters. In fact, ObjectDescriptor is actually implicitly just a GraphicString! The formal specification for an ObjectDescriptor is:

      Discussion

      ObjectDescriptor ::= [UNIVERSAL 7] IMPLICIT GraphicString

      GraphicString is just 0x20 to 0x7E, therefore ObjectDescriptor is just 0x20 to 0x7E.

      Sources: ASN.1: Communication Between Heterogeneous Systems, pages 175-178 The Wikipedia Page on ISO 2022 ISO 2022

      deprecated abstract const @property External external();

      Decodes an EXTERNAL, which is a constructed data type, defined in the International Telecommunications Union's X.680.

      Discussion

      The specification defines EXTERNAL as:

      EXTERNAL := [UNIVERSAL 8] IMPLICIT SEQUENCE { identification CHOICE { syntax OBJECT IDENTIFIER, presentation-context-id INTEGER, context-negotiation SEQUENCE { presentation-context-id INTEGER, transfer-syntax OBJECT IDENTIFIER } }, data-value-descriptor ObjectDescriptor OPTIONAL, data-value OCTET STRING }

      This assumes AUTOMATIC TAGS, so all of the identification choices will be context-specific and numbered from 0 to 2.
